Local Storage Vs. Session Storage

Local Storage :This read-only interface attribute allows access to the Document's local storage object; the information is saved across browser sessions. Similar to session storage, with the difference that local storage data is deleted when the browser window is closed or the page's session expires. The last "private" tab in a browser is cleared when it is closed.

Session Storage :Access to Session Storage objects is made possible by the read-only sessionStorage attribute. The distinction between the two forms of storage is that data in local storage never expires, whereas data in session storage is erased after a page session finishes.

How does JavaScript event loop work ?

Event Loop : The call stack and callback queue tracking is the event loop's main responsibility. If the Call Stack is empty, the Event Loop will retrieve the first event from the queue and put it to the Call Stack, where it will then be executed.
